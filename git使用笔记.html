git使用笔记

git
tools
发布日期:   2020-09-12
更新日期:   2022-03-01
文章字数:   1.3k
阅读时长:   5 分
阅读次数:  

0. git init

初始化本地仓库推送到远程空仓库

$ mkdir NoteBook
$ cd NoteBook
$ git init
$ touch README.md
$ git add README.md
$ git commit -m "first commit"
$ git remote add origin https://gitee.com/nongguangxin/NoteBook.git
$ git push -u origin master

初始化本地仓库拉取远程仓库

$ mkdir NoteBook
$ cd NoteBook
$ git init
$ git remote add origin https://gitee.com/xxx/xxx.git
$ git pull origin master

origin :仓库的名称，可以根据意愿更改。

1. start a working area (开始一个工作区)

1.1 clone — 克隆仓库到一个新的目录

$ git clone https://gitee.com/xxx/xxx.git

git clone 指定分支：

$ git clone https://gitee.com/xxx/xxx.git -b <branch_name>

1.2 init — 创建一个空的Git仓库或重新初始化一个存在的仓库

在github或gitee上创建一个空的仓库后。在本地执行如下操作即可开始一个新的开发。

$ mkdir NoteBook
$ cd NoteBook
$ git init
$ touch README.md
$ git add README.md
$ git commit -m "first commit"
$ git remote add origin https://gitee.com/xxx/xxx.git
$ git push -u origin master

2. work on the current change (处理当前的改动)

2.1 add — 更新索引文件，以追踪文件改动。

$ git add test1
# or add all file
$ git add .

2.2 mv — 移动或重命名文件，目录或链接

$ git mv test1 test

2.3 restore — 还原上一次提交以来对文件的更改

$ git restore test1

恢复git rm 删除的文件。

$ git restore --staged test
$ git restore .

恢复 rm 删除的文件

$ git restore test
# or restore all file
$ git restore .

2.4 rm — 删除文件

$ git rm test

3. examine the history and state (检查历史和状态)

3.1 bisect — 使用二分搜索查找导致错误的提交

3.2 diff — 显示改动

显示上次提交以来的改动

$ git diff HEAD

显示两次提交之间的改动

$ git diff d439cb7b 8a74c825

3.3 grep — 打印匹配的行

$ git grep "xxx"

3.4 log — 显示提交日志

$ git log
$ git -p -2

-p展开显示每次提交的内容差异，-2仅显示最近的两次更新。
--stat仅显示简要的增改行数统计。

3.5 show — 显示各种信息

包括提交信息，修改记录等

$ git show

3.6 status — 显示工作树当前的状态

$ git status

4. grow, mark and tweak your common history（增加，标记和调整你的历史）

4.1 branch — 列出，创建，删除分支

  • -d, –delete
  • -l, –list
  • –show-current

从master分支创建一个新的分支gxnong

$ git branch gxnong master

查看当前所在的分支

$ git branch --show-current

查看远程分支

$ git branch -r
# or show all branch
$ git branch -a

切换到其他分支

$ git switch gxong

或

$ git checkout gxong

将分支push到远程仓库的master分支

$ git push --set-upstream origin master

第一次push需要设定上游分支，第二次就可以直接push

$ git push

如果远程gxnong分支已存在，则可以将当前分支push到远程仓库origin的gxnong分支

$ git push origin gxong

将A分支push到master分支

$ git push origin A:master

强制用本地的代码去覆盖掉远程仓库的代码

$ git push -f origin gxong

删除分支

$ git branch -d gxong

删除远程分支

$ git branch -d -r origin/test3

删除不存在对应远程分支的本地分支

$ git remote prune origin

此时查看登陆远程仓库发现分支还在，使用下面的方法删除。

$ git push origin --delete test3

4.2 commit — 记录改动到仓库

$ git commit -m "message"
$ git commit -a -s -m "mssage"

-a, -add 自动添加已追踪的文件;
git -s 签署sign-of-by

撤销上次提交，重新提交。

$ git commit --amend

4.3 merge — 合并两个或多个开发历史

将分支test3合并到当前分支

$ git merge test3

4.4 reset — 重置当前 HEAD 到指定的状态

$ git reset --hard HEAD^		# 回到上一个版本
$ git reset --hard HEAD^^		# 回到上上一个版本
$ git reset --hard HEAD~20		# 回到20个版本之前

如果想回到某个版本，可以用git reflog查看记录，然后选择回到的版本

$ git reset --hard 8a74c82

4.5 checkout — 从某次提交中新建分支

$ git checkout 38bed85 -b test

4.6 switch — 切换分支

$ git switch test3

4.7 rebase — 修改历史

git rebase 允许大修改历史，谨慎使用。

5. collaborate (协作 )

5.1 fetch — 拉取远程仓库的最新改动

$ git fetch

查看改动

$ git diff FETCH_HEAD

合并改动

$ git merge

5.2 pull — 拉取并合并

其本上等于 fetch + merge

$ git pull

5.3 push — 更新远程库

将改动推送到远程分支

$ git push

5.4 fork 和 pull request

如果想向某个开源项目做贡献或者自己修改某个开源项目，可以将项目fork后，pull到本地，修改完成后如果想贡献自己的代码到源仓库，可以提交pull request请求。

6. 杂项

修改最近一次提交的信息

$ git commit --amend

修改git默认编辑器

$ git config –global core.editor vim

显示文件的修改者，修改时间和修改记录

$ git blame test2

更多详细内容可以查看git帮助

$ git help

远程仓重命名

$ git remote rename origin test

移除远程仓库

$ git remote rm test

补丁

使用git diff 生成unix标准补丁，相关选项查看手册

$ git diff > test.diff

使用git format-patch 生成git标准的补丁，相关选项查看手册

$ git format-patch HEAD^

检查补丁是否能够干净顺利地应用到当前分支中

$ git apply --check test.patch

应用diff补丁

$ git apply test.patch

或应用patch补丁

$ git am test.patch
